## 1. 제품 개요

### 한 줄 요약
Ghost를 헤드리스 CMS로 활용하고 Next.js로 구축하는 라이프스타일/문화 뉴스레터 커스텀 랜딩 사이트

### 해결하는 문제
Ghost 템플릿 기반 운영은 디자인 자유도가 극히 제한적이다. 브랜딩에 맞는 히어로 레이아웃, 애니메이션, 섹션 구성 등을 마음대로 바꿀 수 없어 komki만의 분위기를 만들기 어렵다.

### 핵심 가치
- Ghost의 멤버십/뉴스레터 발송/에디터 기능은 그대로 유지
- 프론트엔드만 완전히 분리 → 디자인 100% 커스텀 가능
- Ghost Content API + Members API를 통해 콘텐츠와 구독자 관리는 Ghost에서 처리

---

## 2. 사용자

### 주요 사용자
- **누구**: 라이프스타일/문화 콘텐츠에 관심 있는 20~40대 독자
- **상황**: 소셜미디어나 검색을 통해 komki 뉴스레터를 처음 접하는 신규 방문자
- **목표**: 뉴스레터가 어떤 콘텐츠인지 파악하고 구독 여부 결정

### 사용자 시나리오
1. SNS에서 komki 콘텐츠 링크를 타고 랜딩페이지 방문
2. Hero 섹션에서 뉴스레터 주제와 분위기를 파악
3. 최신 아티클 미리보기로 콘텐츠 품질 확인
4. 이메일 입력 후 구독 완료 → Ghost가 환영 메일 발송

## 3. 핵심 기능

| 기능 | 설명 | 우선순위 | 복잡도 |
|------|------|----------|--------|
| 커스텀 랜딩페이지 | Hero + 소개 + 최신 글 3개 미리보기 + Footer | P1 (MVP) | 간단 |
| 뉴스레터 구독 CTA | 이메일 입력 폼 + Ghost Members API 연동 | P1 (MVP) | 보통 |
| 아티클 리스트 페이지 | Ghost API 포스트 목록, 태그 필터, 페이지네이션 | P2 | 보통 |
| 아티클 상세 페이지 | 본문 렌더링, OG 메타, SEO, 이전/다음 글 | P2 | 보통 |
| 검색 기능 | 제목/태그 기반 검색 | P2 | 복잡 |
| 유료 멤버십 | Stripe 결제 연동, 프리미엄 콘텐츠 잠금 | P3 | 복잡 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름 — 구독 전환
```
랜딩페이지 방문 -> Hero CTA 클릭 -> 이메일 입력 -> Ghost Members API -> 구독 완료 메시지
```

### 상세 흐름
1. **방문**: `/` 랜딩페이지 로드 (Next.js SSG, Ghost API에서 최신 글 3개 fetch)
2. **탐색**: Hero 섹션 → 소개 섹션 → 최신 아티클 카드 확인
3. **구독**: 이메일 폼 입력 → POST `/api/subscribe` → Ghost Members API 호출
4. **피드백**: 성공 메시지 노출, Ghost가 자동 환영 이메일 발송

---

## 5. 성공 기준

- [ ] 랜딩페이지 Lighthouse Performance 점수 90 이상
- [ ] 이메일 입력 후 Ghost 관리자에 구독자 등록 확인
- [ ] 모바일(375px) + 데스크톱(1280px) 레이아웃 정상 렌더링
- [ ] Vercel 배포 후 외부 URL로 접속 가능

---

## 6. 안 만드는 것 (Out of Scope)

> MVP에서 아래 기능은 만들지 않습니다.

- 유료 멤버십 (Stripe) — Phase 3에서 별도 구현
- 댓글 기능 — 커뮤니티 구축 전략 미확정
- 검색 기능 — Phase 2 이후 콘텐츠 양이 충분할 때
- 로그인/마이페이지 — 구독자 대시보드는 Phase 3
- 다국어 지원 — 현재 한국어 독자 대상
